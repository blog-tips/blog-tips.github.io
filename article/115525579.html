ブログのメタデータをカスタマイズ

ブログのメタデータには、キーワード設定とかブログ説明というのがありますが、最近ウェブマスターツールにログインしてみたら、以前よりもコンテンツ分析の問題が細分化されているような気がします。

メタタグの重複とかは正直、無料ブログではどうしようもないというのもあるので、ほったらかしにしてたんですが、それに加えて長すぎるものや短すぎるものも問題みたいなんで修正してみようかなと思うんです。

タイトルや説明など問題のあるメタデータ

メタデータのなかでも、グーグルは keywords meta タグのメタデータはあまり重要視しないようなんで、ウェブマスターツールでも descriptions とタイトルタグの問題ぐらいしかのってないようです。

ブログ説明もタイトルタグも、長すぎる場合や短すぎるもの、重複するものはどれも問題があるみたいです。

メタタグは検索結果の順位に関係ないようなことも聞いたことがあります。

けれども、的確なブログ説明やタイトルタグを記入することで検索結果でのクリック率が上昇すると、やはり順位も少しは上昇するのではないかなと思うんです。

メタタグのブログカスタマイズ方法

無料ホームページなど html を直接記入することができるウェブサイトの場合、個別にまったく違うブログ説明を入力することができますが、無料ブログの場合はすべてを違うように記入するのはまず無理かと思います。

特に、過去ログやカテゴリページでは、どうしてもタイトルタグやブログ説明が重複しがちです。

過去ログは検索結果でヒットすることはあまりないので、インデックスさえされればいいかなって思うんで、ぶっちゃけメタタグはなくてもいいかなとも思うんですが、個別エントリーのページは、なるべく違う感じになるようにブログカスタマイズしたいものです。

シーサーブログの場合、こんな感じにするといいっぽいんですが、どうなんでしょうか。

<meta name="description" content="<% if:extra_title %><% extra_title %>。<% /if %><% blog.description | nl2br | tag_strip %>" />

個別エントリーにはブログ説明の前に記事タイトルが追加される形です。

ＭＴブログや無料ホームページはメタタグ自由

無料ブログレンタルの場合、メタタグの重複や長短で問題がでてきてしまうことが多いですので、いっそのこと削除した方がいいのかなという気もしてきます。

けれども、無料ホームページの場合はまったくの自由ですし、ＭＴブログの場合もカスタマイズしやすいので、なるべくなら記入しておきたいものです。

ＭＴブログのメタタグの場合、キーワード入力には <$MTEntryKeywords$> などをいれておけばいいのかなと思います。

エントリーのメタタグ説明は、もしタグ機能を使っていないようであれば、mtif による場合わけが必要ですが、そこにメタタグ description を入力すれば、個別に自由設定できるのでおすすめです。

<meta name="description" content="<MTEntryIfTagged><MTEntryTags glue=", "><$MTTagName$></MTEntryTags></MTEntryIfTagged>" />

シーサーブログでもキーワードのメタタグをはずして、キーワード入力欄にエントリーのブログ説明を入力してもいいんじゃないかなとも思います。

グーグルではキーワードのメタタグはそれほど重要視しないようですし、ヤフーからのアクセスがない場合はメタタグ description だけでもいいんじゃないかなって気もします。

実際に機能するのかどうかは確認してないんですが、たぶん、独自タグとしてはこんな感じになるかと思います。

<meta name="description" content="<% extra_keywords | oneline | tag_strip %>" />

ただ、キーワードのメタタグをはずすかといったら、私ははずしませんが、どちらかというと、やはり keywords よりも description のメタデータの方が重要な感じがします。

私はネット上で検索するときに、検索順位よりも description メタタグで判断することがけっこう多いです。

ブログ訪問者さまに入らぬお手間をかけさせないためにも、description メタタグは適切に記入したいものです。
